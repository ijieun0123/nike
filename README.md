# 🚀 나이키 리뉴얼 프로젝트

본 프로젝트는 나이키 웹사이트의 현대화 및 사용자 경험(UX) 개선을 위한 리뉴얼 작업의 일환으로 진행되었습니다.

https://ijieun0123.github.io/nike/

---

### 💻 기술 스택 (Tech Stack)

| 분류           | 기술                    | 역할                                                    |
| :------------- | :---------------------- | :------------------------------------------------------ |
| **프레임워크** | React (with Vite)       | 컴포넌트 기반 UI 구축                                   |
| **언어**       | TypeScript              | 정적 타입 검사                                          |
| **스타일링**   | Styled-Components, SCSS | 동적 스타일링 및 SCSS 문법 활용                         |
| **슬라이더**   | Swiper                  | 아이코닉 슈즈 및 멤버십 카드, 트렌딩 카드 슬라이더 구현 |
| **애니메이션** | AOS                     | 스크롤 시 요소 노출 애니메이션 적용                     |

---

### 🛠 핵심 기술 구현 상세

<details>
<summary><b>1. TypeScript를 통한 "Type-Safe UI" 구현</b></summary>
<br>

-   **엄격한 데이터 관리:** 모든 UI 데이터와 Props에 Interface를 적용하여 데이터 구조를 엄격하게 관리했습니다.
-   **오류 차단 및 생산성 향상:** 협업 시 발생할 수 있는 데이터 타입 불일치 오류를 사전에 차단하고, 자동 완성 기능을 통해 개발 생산성을 높였습니다.
</details>

<details>
<summary><b>2. Styled-components 기반의 동적 컴포넌트 설계</b></summary>
<br>

-   **다양한 속성을 제어하는 Props 시스템:** `RoundBtn` 컴포넌트에서는 `btnName`과 `color` props를 통해 버튼의 텍스트와 테마를 제어했습니다. 또한 `TitleBox`에서는 `color`, `title`, `subtitle1`, `subtitle2` 등 상세한 props를 설계하여 타이틀의 색상과 위계별 텍스트 내용을 동적으로 관리했습니다.
-   **컴포넌트 단위 스타일 캡슐화:** 각 컴포넌트의 스타일을 내부에서 정의하여 외부 스타일과의 충돌을 방지하고 코드의 독립성을 확보했습니다.
-   **유지보수 효율 극대화:** 공통 UI 요소를 독립적인 부품 단위로 분리하여, 스타일 변경 시 해당 컴포넌트만 수정하면 프로젝트 전체에 즉각 반영되는 효율적인 구조를 구축했습니다.
</details>

<details>
<summary><b>3. "Pixel-Perfect"를 지향하는 정밀한 UI 구현</b></summary>
<br>

-   **수치 기반의 완벽한 재현:** 피그마 시안의 절대 수치를 분석하여 브라우저 환경에 맞는 상대 비율(%)로 정밀하게 환산했습니다. 어떤 해상도에서도 디자인 의도가 오차 없이 유지되도록 구현했습니다.
-   **유연하고 견고한 반응형 설계:** 주요 중단점(768px, 1024px, 1440px)별로 폰트 크기, 자간, 여백 등을 재설정하여 단순 유동형 레이아웃의 한계를 넘어 최적의 시각적 밸런스를 유지했습니다.
-   **전략적 콘텐츠 노출 관리:** `hide_on_mobile` 클래스 등을 활용해 기기별 최적화된 콘텐츠만 노출함으로써 시각적 밸런스가 완벽한 인터페이스를 제공하는 데 집중했습니다.
</details>

<details>
<summary><b>4. 사용자 경험을 고려한 인터랙션 및 라이브러리 최적화</b></summary>
<br>

-   **콘텐츠별 유연한 슬라이더 배치:** `Iconic`, `Membership`, `Trending` 등 다양한 섹션의 슬라이더에 `slidesPerView: "auto"` 설정을 적용했습니다. 기기별로 슬라이드 개수를 강제로 고정하지 않고 해상도에 맞춰 카드가 유연하게 노출되도록 설계하여 콘텐츠가 어색하게 잘리는 현상을 최소화했습니다.
-   **브랜드 무드에 맞춘 인터랙션:** `AOS.js`를 활용하여 콘텐츠 등장 시점에 적절한 지연 시간을 부여함으로써, 사용자의 시선 흐름에 맞춘 리듬감 있는 브랜드 경험을 연출했습니다.
</details>

<details>
<summary><b>5. SCSS 시스템 설계 및 모듈화 작업</b></summary>
<br>

-   **디자인 시스템의 변수화 (\_variables.scss):** 컬러, 폰트 규격, 행간, 자간 등을 시스템 변수로 정의하여 프로젝트 전반에 걸쳐 일관된 UI 구현 및 유지보수 효율을 높였습니다.
-   **스타일 패턴의 재사용 (\_mixins.scss):** 반복되는 CSS 속성을 믹스인으로 모듈화하여 코드 중복을 최소화했습니다.
-   **반응형 가이드 및 레이아웃 설계 (\_common.scss):** 정밀한 해상도별 비율 계산을 통해 `.container`를 규격화하고, 텍스트 위계에 따른 공통 클래스를 관리하여 스타일 파편화를 방지했습니다.
-   **구조적 스타일 관리:** 스타일 시트를 `reset`, `variables`, `mixins`, `common`으로 분리하여 관리함으로써 유지보수가 용이한 구조를 지향합니다.
</details>

### 🔍 트러블슈팅

<details>
<summary><b>1. 반응형 환경에 따른 복합 그리드 레이아웃 및 시각적 위계 재설계</b></summary>
<br>

-   **Challenge:** `brand_section`에서 이미지와 텍스트가 해상도에 따라 완전히 다른 배치를 가져야 했습니다. 모바일에서는 수직 위주의 단순한 나열이 필요했지만, 데스크탑에서는 제품 이미지가 텍스트를 감싸거나 특정 위치에 겹쳐야 했고, 섹션마다 좌우 정렬이 교차되는 등 정교한 위치 제어가 요구되었습니다.
-   **Solution:**
    -   **Flexbox 기반의 유연한 흐름 제어:** 모바일의 `column` 구조를 데스크탑에서 `row`로 전환하고, `order` 속성을 활용하여 마크업 순서를 변경하지 않고도 텍스트와 이미지의 시각적 좌우 배치를 자유롭게 조정했습니다.
    -   **정밀한 좌표 및 음수 마진 활용:** 이미지와 텍스트 박스 간의 의도적인 겹침 효과를 위해 `position: absolute`와 음수 마진(`margin-top`)을 적절히 혼합했습니다. 특히 퍼센트(`%`) 단위의 너비와 좌표를 활용해 브라우저 크기가 변하더라도 요소 간의 상대적인 간격이 유지되도록 설계했습니다.
    -   **콘텐츠 최적화 이미지 렌더링:** `background-size: contain/cover`를 섹션별 이미지 특성에 맞춰 다르게 적용하고, 고정 수치(px)와 가변 수치(%)를 전략적으로 혼합하여 이미지 왜곡 없이 개방감 있는 브랜드 섹션을 구현했습니다.
-   **Result:** 모바일에서의 높은 가독성과 데스크탑에서의 역동적인 브랜드 이미지를 동시에 확보했습니다. 특히 소수점 단위의 세밀한 위치 조정을 통해 어떤 해상도에서도 시안과 일치하는 'Pixel-Perfect' 레이아웃을 완성했습니다.
</details>
